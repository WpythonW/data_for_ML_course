# Сценарий видео — ML Data Pipeline с Claude Code

> Запись демонстрации полного пайплайна подготовки ML-датасета:
> поиск → чистка → авторазметка → active learning.
> Видео: https://drive.google.com/drive/folders/1lw1AV9gAPa6Vb1bh69g7H-8kpW4edqnU?usp=sharing

---

## Задача

Найти датасет для **бинарной классификации текстов** (английский, 1 000–10 000 строк),
пройти все 4 этапа ML-пайплайна и показать результат.

---

## Этап 0 — Setup

- Открываем проект в Claude Code: `claude .`
- Запускаем пайплайн одной командой:
  ```
  /pipeline датасет для классификации текстов! Язык - англ или рус, количество строк от 1к до 10к, но не больше
  ```
- Claude читает `skills/data_pipeline.md` и идёт по нему строго по шагам.
- Проверяет наличие `.env` (HuggingFace, Kaggle, OpenRouter) — все ключи на месте.

---

## Этап 1 — Dataset Collector

**Что происходит:**
- Запускается поисковая волна: HuggingFace + Kaggle по запросам
  `"text classification dataset"`, `"sentence classification"`, `"review classification"` и др.
- Найдено 132 сырых датасета → BM25 ранжирование → OpenRouter фильтрация → топ-10 кандидатов.

**HITL Checkpoint #1** — Claude показывает таблицу из 5 финальных датасетов:

| # | Датасет | Строк | Классы |
|---|---------|-------|--------|
| 1 | pooja-gani/qasper-sentence-classification | 3 733 | только 1 класс ❌ |
| 2 | kdercksen/pragtag_sentence_classification | 4 911 | 6 классов |
| **B** | **akoukas/chatgpt-classification-sentence-level** | **7 344** | **human / AI** |
| ... | ... | ... | ... |

- Первый выбор (#1) оказался однокласcовым — Claude сам обнаружил проблему и предложил замену.
- Пользователь выбирает **датасет B**: human vs AI-generated sentences, 7 344 строки.
- Датасет скачивается, приводится к unified schema: `text, label, source, collected_at`.
- Сохраняется в `data/raw/unified.csv`.

---

## Этап 2 — Data Detective

**Что происходит:**
- Запускаются 4 детектора: пропуски, дубликаты, выбросы, дисбаланс.

**Результаты:**

| Проблема | Кол-во | Серьёзность |
|----------|--------|-------------|
| Пропуски | 0 | нет |
| Дубликаты | **47 строк** | низкая |
| Выбросы | 0 | нет |
| Дисбаланс | 55% / 45% | низкая |

**HITL Checkpoint #2** — пользователь выбирает стратегию: `remove duplicates`.

- Удалено 47 дублей: 7 344 → **7 297 строк**.
- Сохраняется в `data/cleaned.csv`.

---

## Этап 3 — Annotation Agent

**Что происходит:**
- Модель `cross-encoder/nli-MiniLM2-L6-H768` (~120 MB) размечает все 7 297 строк
  через zero-shot классификацию: **human** vs **AI-generated**.

**HITL Checkpoint #3** — подтверждение настроек → пользователь отвечает `да`.

**Результаты:**
- Confidence среднее: **0.80**
- Флаги для ручной проверки: **2 271 строк (31%)** — ниже порога 0.75
- Экспорт: `data/labelstudio_import.json` — готово для загрузки в LabelStudio
- `data/low_confidence.csv` — строки под ручную разметку
- `data/annotation_spec.md` — спецификация классов

---

## Этап 4 — Active Learner

**Что происходит:**
- LogisticRegression + TF-IDF обучается на 50 стартовых примерах.
- Стратегия **entropy** итеративно выбирает 5 × 20 = 100 наиболее неопределённых примеров.
- Параллельно — **random** baseline для сравнения.

**HITL Checkpoint #4** — пользователь отвечает `да`.

**Результаты (150 меток):**

| Стратегия | F1 @ 150 меток |
|-----------|----------------|
| Entropy   | 0.6558 |
| Random    | 0.6703 |

- На этом датасете (бинарная задача, умеренный пул) random и entropy дают близкие результаты —
  это ожидаемо: бинарные задачи с балансированными классами меньше выигрывают от uncertainty sampling.
- График кривых обучения: `data/learning_curve.png`

## Итог

Полный пайплайн от поискового запроса до кривых обучения — **без написания кода вручную**.
Claude Code оркестрирует все 4 агента, останавливается на checkpoint'ах и ждёт решения человека.

```
/pipeline <задача>  →  данные  →  чистка  →  разметка  →  active learning
```

**Готовые файлы:**
- `data/cleaned.csv` — очищенный датасет с оригинальными метками
- `data/labeled.csv` — датасет с NLI авторазметкой
- `data/annotation_spec.md` — спецификация классов
- `data/labelstudio_import.json` — импорт для LabelStudio
- `data/learning_curve.png` — кривые обучения
